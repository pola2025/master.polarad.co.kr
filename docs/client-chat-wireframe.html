거래처 채팅을 요청건별 업무 관리로 분리

고객은 초대 링크로 들어와 등록 이메일 OTP 인증 후 요청건을 생성합니다. 관리자는 거래처별 요청건을 접수하고 제목, 요약, 체크리스트를 정리한 뒤 완료 처리하면 해당 요청건 채팅을 종료합니다.

관리자: /chats 고객: chat.polarad.co.kr/{email local-part} 이메일 OTP 인증 초대: 메일 또는 문자 선택 완료 요청건 채팅 종료 R2 파일 10MB 제한
  • 링크만으로 방 데이터가 열리지 않습니다. 공개 API는 OTP 세션 쿠키가 있어야 응답합니다.
  • 요청건 제목은 관리자가 지정하고 목록에는 요청일시가 항상 자동 prefix로 표시됩니다.
  • 요청건수는 관리자가 접수한 건 기준으로 관리합니다. 미접수 상담은 별도 배지로 분리합니다.
  • 관리자가 파일을 다운로드하면 R2 객체를 삭제하고 첨부 상태를 삭제로 표시합니다.

관리자 대시보드: 거래처 → 요청건 → 대화

요청건 제목/상태/요약/체크리스트를 관리하고 완료 시 대화를 잠급니다.

보안/파일 처리 흐름

외부 노출을 줄이고, 파일은 private R2 객체로만 처리합니다.

Backend contract
1

초대 발송

관리자가 이메일 또는 문자로 chat.polarad.co.kr/{slug} 링크를 발송합니다.

2

OTP 인증

등록 이메일과 일치할 때만 인증번호를 발송하고, 성공 시 httpOnly 세션 쿠키를 발급합니다.

3

요청건 생성/접수

고객 요청은 draft로 생성되고, 관리자가 제목/요약/체크리스트를 저장하며 접수 처리합니다.

4

완료 후 종료

상태가 완료 또는 취소면 메시지/첨부 API가 추가 입력을 차단합니다.

5

파일 업로드

10MB 이하만 허용하고 R2 binding CHAT_FILES에 저장합니다.

6

관리자 다운로드

관리자가 다운로드하면 R2 객체 삭제 후 downloaded_at/deleted_at을 기록합니다.

7

수신확인

관리자 열람 시 read_by_admin_at, 고객 열람 시 read_by_client_at 저장.

8

운영 알림

고객 메시지는 Telegram 알림으로 관리자에게 전달됩니다.
